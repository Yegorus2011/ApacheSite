About my training

Here you can find information about the ways in which I have trained myself, as I became more familiar with certain services and techniques that are used by people in system administration. I will explain my progress step by step.

Chapter 1: technical difficulties...

So the first thing that I decided to do was become acquainted with the basic structure of a typical PC. I bought a really old HP dc7800 Compaq model. Initially, it didn't work at all (as you're about to see).

However, I didn't stop. Instead, I tried to fix the problem in many ways. I replaced the CMOS battery (responsible for proper BIOS functioning), replace the memory cards (2GB to 4GB), replace HDD with SSD, change the boot order... but none of it worked.

In fact, things became even worse than they were before. My PC started to malfunction because of incompatibility issues, as can be seen based on the image below.

This is what made me realize that the problem had to do with the motherboard. Apparently, there was an actual smart card reader feature installed on the PC before, but the previous owner simply forgot to disable it. This is why I had to ask one of my good aquaintences for help, so he could use his chip programmer to overwrite the hardware. AND IT WORKED.

Following that, I simply connected the new PC to my network. Speaking of which...

Chapter 2: Enter Windows, Linux servers (and others...)

Of course, simply understanding how a PC works wouldn't be enough for me, I also had to "dig" into how servers work. Like a lot of people, I started with Windows Server. And I decided to integrate that with things like AWS, Azure and Hyper-V.

First, I created 2 virtual machines - One for the server (Server_lab_2022(DC)) and another one for the client (CLIENT1). To make sure that the two machines can connect and the DHCP worked properly, I had to configure two networks (one internal (static), one for the internet (dynamic)) Then I generated the code to fill up my directory with user profiles and such. Stage 5 (Life after Linux) This is the stage at which I had finally decided to start taking Linux seriously. Of course, like most people, I had to start with learning the commands, primarily with the help of the many public resources from the internet:

Stage 5.1 (what are dockers) I started to become acquainted with Linux services (not just commands) by trying to understand docker. A docker is essentially a virtual machine for applications. Except it's different in terms of how it uses resources of your PC. Virtual machines typically have their own kernel, dockers do not.

Stage 5.2 (Linux basics) And this is where I started to study actual integrated Linux services. As always, I began with the basics. Specifically, I configured samba on my old PC, turning it into a file system to be used by local users.

Stage 5.3 (More basics) After I created the Samba share, I decided to create some directories (mnt/sda2,3), make them available (using chmod), and then mount file systems on top of them. Then I pooled them together with the help of MergerFS. This gave me something similar to an NAS server.

Stage 5.4 (Print server) Next, I made a basic print server, which distributes the printer as a common device, allowing multiple users to access the service. In other words, people use it like it's connected to them directly.

Stage 5.4 (Plex Bonus) As a bonus, I wanted to configure Plex on one of my PCs. In case you don't know, Plex is a type of software that takes your data (especially visual data), and then compartmentalizes it in a convinient way, like on a website.
